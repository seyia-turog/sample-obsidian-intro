## Overview

The **Standing Instruction (Mandate) Module** manages recurring payments and automated financial instructions initiated by users.

It handles creation, modification, retrieval, and deactivation of mandates, integrating with Identity, CBA, CRM, and Notification Workers to ensure secure, accurate, and compliant mandate lifecycle operations.

## Technical Dependencies

### Adapter & Processor Dependencies

| Adapter / Processor        | Business Purpose |
| -------------------------- | ---------------- |
| Identity Adapter           | Validates user session, permissions, and mandate access. |
| CBA Adapter                | Creates, updates, and reads mandate data from the Core Banking Application. |
| CRM Adapter                | Logs mandate activity and synchronizes customer mandate history. |
| Util Worker (Messages)     | Sends mandate creation, update, and cancellation notifications. |
| Processor (Documents)      | Stores mandate terms or changes as structured documents. |
| User Settings Utility      | Configures payment preferences and mandate behavior. |
